### Вопросы с которыми столкнулся


1. Хочу обратиться к базе данных, но не хочу писать query запросы в бизнес логике.
2. При проектировании базы данных столкнулся с тем, что id в спецификации не числа, а строки.
3. Также, при проектировании столкнулся, что команда, и пр имеют в своей структуре массив пользователей.
4. Как типобезопастно обращаться к спецификации.

### Ответы, к которым пришел

1. Нашел библиотеку sqlc, которая генерирует типобезопасный код.
2. Делаю отдельное поле ID, которое будет ```PRIMARY KEY```, и дополнительно ```UNIQUE NOT NULL``` поле, которое будет обозначать ID
3. У участника добавил поля с ID team и pr. Также дополнительно создал индекс на эти поля, для более быстрого поиска.
4. Нашел генератор кода openapi-generator и сгенерировал код командой __№1__ 


#### №1
```bash
docker run --rm \
   -v $PWD:/local openapitools/openapi-generator-cli generate \
   -i /api/openapi.yaml \
   -g go-gin \
   -o /internal/gen/go

```
